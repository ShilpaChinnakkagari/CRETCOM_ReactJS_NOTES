I. State
---------
    State is a built-in React Object used to store data that can change over time when the state changes , React automatically
    update (re-rendering the content) the UI 
    --> stores one state object to another state object.
    --> stores two states(current + next)

syntax:
-------
import {useState} from "react";

function ComponentName() {
    const [stateName, setStateFunc] = useState(intialValOfstateName);

    return(
        <div>
            {/* JSX */}
        </div>
    );
}
 
state = current value
setState = Function to update the State
useState() = creates the state
intialValOfstateName = initial value of state

Example:
----------
stateMgtCount.jsx
-----------------

import { useState } from "react";

export function Count(){
    const [count, setCount] = useState(0);
    
    return(
        <div>
            <h2>Count : {count} </h2>
            <button onClick = {() => setCount(count+1)}> Increment </button>

            <br/>
            <br />
            <button onClick = {() => setCount(count-1)}> Decrement </button>
        </div>
    )
}

Problem: Why my method does not work for storing the previous state value?

Code:
-------
const [text, ChangeText] = useState("Shilpa");
const temp = text;

<button onClick={() => ChangeText("Hansika")}>
    Click
</button>

<button onClick={() => ChangeText(temp)}>
    Reset
</button>


Why it fails:
------------------
Initially:
text = "Shilpa"
temp = "Shilpa"

When Click button is pressed:

ChangeText("Hansika")

State updates:

text = "Hansika"


After state update, React re-renders the component.
During re-render, the component function executes again:

const temp = text;

Now:
------
text = "Hansika"
temp = "Hansika"


The old value of temp ("Shilpa") is lost.

So when Reset button is clicked:

ChangeText(temp)

actually becomes:

ChangeText("Hansika")

because temp currently stores "Hansika".


Important Concept:
---------------------
Normal variables inside a React component are recreated on every render.

Example:
----------
const temp = text;

This does not permanently store the value.

To store values between renders, use:

1. useState()
   - Stores data and updates UI

2. useRef()
   - Stores data without causing re-render


Remember:

Normal variable (let/const)  → Data lost after re-render
useState                    → Data persists and updates UI
useRef                      → Data persists without re-render


Rule:
Never use normal variables to store previous state values in React components.

Q8. Write a JSX Script to perform state functionalitites using two scenarios 
    (i) single click : The container color has to change to "violet"
    (ii) double click : The container color has to go initial state


import React, { useState } from "react";

function ColorChange(){

    // Initial state
    const initialColor = "white";
    const [color, setColor] = useState(initialColor);


    // Single click function
    function changeColor(){
        setColor("violet");
    }


    // Double click function
    function resetColor(){
        setColor(initialColor);
    }


    return(
        <div>

            <h1>State Functionality</h1>

            <div
                style={{
                    width:"300px",
                    height:"200px",
                    border:"2px solid black",
                    backgroundColor: color,
                    textAlign:"center",
                    padding:"20px"
                }}

                onClick={changeColor}

                onDoubleClick={resetColor}
            >

                <h2>Click / Double Click</h2>

                <p>
                    Current Color : {color}
                </p>

            </div>

        </div>
    );
}

export default ColorChange;
